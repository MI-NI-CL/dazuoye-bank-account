# dazuoye-bank-account
初学的c++课程大作业，实现银行ATM系统最重要的功能，最后使用AI优化过，故这版不作为正式作业提交，仅供学习
================================================================================
                        ATM 银行系统 —— 项目说明文档
================================================================================

一、项目概述
--------------------------------------------------------------------------------
  本程序是一个基于命令行的 ATM 银行模拟系统，使用 C++17 开发。
  支持用户注册/登录、个人信息管理、多类型银行卡管理、存取款操作、
  利息计算、活跃度统计等功能。

  项目采用多文件结构，包含以下模块：
    main.cpp              - 程序入口
    system/               - 系统核心：主菜单、登录注册、卡操作调度
    User/                 - 用户类：个人信息管理与文件持久化
    cardtype/             - 银行卡类族：抽象基类 + 三种具体卡类型
      card.hpp/cpp        - 银行卡抽象基类，定义统一接口
      Debit.hpp/cpp       - 储蓄卡
      Credit.hpp/cpp      - 信用卡
      Comp_card.hpp/cpp   - 复合卡


二、编译方法
--------------------------------------------------------------------------------
  使用 g++ (MinGW-w64) 编译，在项目根目录执行：

    g++ -std=c++17 -o main.exe main.cpp system/system.cpp cardtype/card.cpp ^
        cardtype/Debit.cpp cardtype/Credit.cpp cardtype/Comp_card.cpp ^
        User/User.cpp -lws2_32 -lshell32

  运行：直接双击 main.exe，或在命令行中输入 .\main.exe


三、功能说明
--------------------------------------------------------------------------------

  主菜单:
    1. 登录    - 输入用户名密码，3次失败后账户锁定
    2. 注册    - 创建新用户，信息存入 users/ 目录
    3. 统计    - 显示所有用户的平均登录次数（活跃度）
    4. 退出

  用户菜单（登录后）:
    1. 显示个人信息
    2. 修改密码
    3. 修改住址
    4. 修改电话
    5. 查看所有银行卡
    6. 添加银行卡（储蓄卡/信用卡/复合卡）
    7. 注销银行卡
    8. 操作银行卡（存款/取款/查询余额/计算利息）
    9. 查看个人活跃度
    0. 退出登录


四、银行卡类型说明
--------------------------------------------------------------------------------

  (1) 储蓄卡 (Debit)
      - 不允许透支，余额必须 >= 取款金额
      - 存款余额享受利息收益
      - 利率: 0.0001/分钟（即余额的 0.01% 每分钟）

  (2) 信用卡 (Credit)
      - 允许透支，额度上限 114514 元
      - 正常余额无利息，透支后产生罚息
      - 罚息率: 0.0001/分钟（即透支金额的 0.01% 每分钟）

  (3) 复合卡 (Comp_card)
      - 兼具储蓄和信用功能
      - 正余额时享受高利率收益: 0.6/分钟
      - 透支时产生罚息: 0.0001/分钟
      - 额度上限 114514 元


五、利息计算机制（重要特性）
--------------------------------------------------------------------------------

  本系统的利息/罚息计算基于系统实时时间，而非模拟时间。

  计算原理：
    每次对卡片进行操作（存款、取款、查询）时，系统会获取当前操作系统
    的时间戳，与卡片记录的"上次结息时间"进行对比，计算两者的时间差，
    再根据时间差和对应利率/罚息率计算应得的利息或应扣的罚息。

  具体公式：
    储蓄卡利息 = 余额 x 0.0001 x 经过的分钟数
    信用卡罚息 = 透支金额 x 0.0001 x 经过的分钟数
    复合卡利息 = 余额 x 0.6 x 经过的分钟数  (正余额时)
    复合卡罚息 = 透支金额 x 0.0001 x 经过的分钟数  (透支时)

  这意味着：
    - 两次操作间隔越久，累计的利息/罚息越多
    - 程序关闭后时间仍在流逝，下次登录操作时会一次性结算所有累积利息
    - 数据在退出登录时持久化保存，不会丢失


六、数据持久化
--------------------------------------------------------------------------------

  用户数据存储于 users/ 目录，每个用户一个 .txt 文件，格式为 key=value。
  银行卡数据存储于 cards/ 目录，文件名格式为 用户名_卡号.txt。
  卡片文件格式: 卡类型|卡号|余额|上次结息时间

  所有数据在退出登录时自动保存，下次登录时自动加载。


七、彩蛋说明
--------------------------------------------------------------------------------

  本程序包含若干彩蛋，在用户输入异常数值时触发：

  (1) 取款金额彩蛋
      在卡操作菜单中取款时，如果输入的取款金额超过 19 位数字
      （超出 double 精度范围），系统会调用 Windows 图片查看器
      弹出 caidan.png 图片，取款操作失败。

  (2) 复合卡超额彩蛋
      在复合卡取款时，如果金额超过 long long 类型最大值
      （约 9.22 x 10^18），同样会弹出彩蛋图片。

  (3) 其他趣味提示
      存款或取款 0 元时会触发特殊回复，输入负数金额也有对应吐槽。
